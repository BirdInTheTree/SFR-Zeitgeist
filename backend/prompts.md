# LLM Quality Gate — промпты

Тестировано в сессии eed62a46 (апрель 2026). Два варианта сравнивались на 60 кандидатах из данных 1 апреля.

## Промпт A (отвергнут)

Размытая инструкция, без примеров, без формата ответа:

```
Из этих 50 фраз убери: техническую разметку, бытовые слова без
новостного значения, обрывки фраз. Оставь только те что описывают
события, темы, людей, организации, места. Верни 36.
```

**Проблемы:** нет чёткого определения "мусор", нет контекста (spike, programs), нет формата ответа.

## Промпт B (выбранный)

Бинарный фильтр с примерами и контекстом:

```
Ты фильтр для дашборда "О чём говорит Швейцария сейчас".
Для каждой фразы ответь: 1 (zeitgeist) или 0 (шум).

Zeitgeist: события, люди, места, организации, явления о которых
говорят новости. "Bosnien-Herzegowina", "Kunsthaus Zürich", "Apple".

Шум: бытовые предметы, разговорные слова, технические маркеры,
фрагменты фраз без контекста. "schönes Kleid", "Doofes", "6 Monaten".

Фразы:
1. Bosnien-Herzegowina (spike=8600, programs: 10vor10, Tagesschau)
2. schönes Kleid (spike=300, programs: Frühling, SRF bi de Lüt)
...
```

**Почему лучше:**
- Бинарный ответ (1/0) — легко парсить
- Примеры zeitgeist vs шум — LLM понимает границу
- Контекст (spike, programs) — LLM видит откуда фраза
- 60 кандидатов → фильтр → 36 — запас на отсев

**Модель:** Claude Haiku (cost-effective, F1=0.154 — 2-е место по Ben Mansour et al. 2025)

## Pipeline (полный, выбранный)

```
spaCy noun phrases + NER
→ spike (24h vs 14 дней)
→ фильтр ≥2 editorial units
→ greedy dedup (cosine < 0.6)
→ rule-based cleanup (regex, числа, single ADJ)
→ топ-60 кандидатов
→ LLM quality gate (промпт B, 1/0 per phrase)
→ первые 36 прошедших → сетка 6×6
```

## Результат теста

На 60 кандидатах 1 апреля оба промпта дали похожий топ-24 (совпадение).
Разница в хвосте (25-36): промпт B чище убирал бытовой мусор.
Итог: 33 чистых фразы из 60 кандидатов.

## Rule-based пре-фильтр (до LLM)

- `SWISS TXT`, `Accessibility Services`, `Untertitelung` → regex
- Фразы начинающиеся с числа (`6 Monaten`) → regex
- Одиночные прилагательные (`Doofes`) → POS filter
- Фразы длиннее 4 слов → обычно парсинговый мусор
